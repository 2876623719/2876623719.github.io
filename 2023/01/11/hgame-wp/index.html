神秘的海报

海报如图所示

经过尝试发现为最低有效位LSB隐写，通过kali工具zsteg（命令zsteg xxx.png）

可得到一半flag，另一个半在链接所给音频中，且使用steghide隐写，密码为6位数字（接下来正常操作应该使用）我直接猜123456正好猜对了

第二周

Sign In Pro Max![](hgame-wp/Sign In Pro Max/1.png)

附件内容如下：

Part1, is seems like baseXX: QVl5Y3BNQjE1ektibnU3SnN6M0tGaQ==
Part2, a hash function with 128bit digest size and 512bit block size: c629d83ff9804fb62202e90b0945a323
Part3, a hash function with 160bit digest size and 512bit block size: 99f3b3ada2b4675c518ff23cbd9539da05e2f1f8
Part4, the next generation hash function of part3 with 256bit block size and 64 rounds: 1838f8d5b547c012404e53a9d8c76c56399507a2b017058ec7f27428fda5e7db
Ufwy5 nx 0gh0jf61i21h, stb uzy fqq ymj ufwyx ytljymjw, its’y ktwljy ymj ktwrfy.

part1，先后使用base64，base58，base32，得到结果f51d3a18

part2，part3，part4，是md5，hash解密，推荐网站：https://hashes.com/en/decrypt/hash

得到结果分别是f91c，4952，a3ed

part4，是凯撒密码，通过CTFCrakTool工具破解为

Part5 is 0bc0ea61d21c, now put all the parts together, don’t forget the format.

至此得到flag：hgame{f51d3a18-f91c-4952-a3ed-0bc0ea61d21c}

crazy_qrcode

打开附件，里面有一张二维码和加密的图片文件，要先从二维码中获取密码。直接扫描二维码，扫不出来。

需要对二维码进行修复，进入工具网站https://merricx.github.io/qrazybox/

首先直接暴力解码，看看有没有密码，结果没有

再修改容错，将容错修改为最高H4，修改后扫码获得密码，解压图片文件包。

打开是很多拆散的图片和一个无后缀文件，因为第0张和第4张是一样的，看起来都是左下角的，推测图片拼接前可能需要旋转。那么记事本里的数字很可能就是宣传的角度，0,1,2,3分级别是顺时针转0,90,180,270度。

将图片在ppt中按顺序旋转拼接获得可以扫的二维码，得到flag

身份证明，添加请求头User-Agent:Cute-Bunny

这里根据响应头来看

响应头添加了cookie code=guest，所以这一步应该是发包时添加cookie cookie:code=Vidar

添加请求头refer:bunnybunnybunny.com

XFF伪造头部添加X-Forwarded-For:127.0.0.1。以下是添加的所有头部

BP添加数据

得到flag

第二周：

Git Leakage

题目给出提示是git泄露，disearch扫一下，果然是git泄露。

直接用githack kali命令:python2 GitHack.py url/.git/，扫出的结果保存在dist文件中，得到flag。
